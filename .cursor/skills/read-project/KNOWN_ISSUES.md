# 已知问题

## v1 网页版历史问题（必须规避）

记录在 `xq/需求.md` §4。参考图：`xq/e7c16ec74c6f355a4ac6f2ee3d24a97a_compress.jpg`。

### BUG-V1-01：保存笔记后阅读位置重置

**现象**：添加笔记并保存后，阅读器跳回章节开头。

**可能原因**：保存笔记时重新渲染章节，或重置了 epub.js 的阅读位置。

**预防措施**：
- 打开笔记弹窗前先保存进度锚点，关闭后恢复
- 保存笔记时不要重新挂载阅读器组件
- 在 store 中更新笔记，避免触发整章重载

### BUG-V1-02：长按选区布局错乱

**现象**：长按选区出现错位蓝色高亮，与行间注释叠在一起（见「问题1」截图）。

**可能原因**：epub.js 与浏览器原生选区 API 和自定义叠加层冲突。

**预防措施**：
- 在逐词 `<span>` 上实现自定义选区，不用浏览器默认选区
- 注释层设 `user-select: none`；仅在「添加笔记」模式才开启选区
- 不要依赖跨 iframe 边界的 Selection API

### BUG-V1-03：进度未保存 / 无限加载

**现象**：重新打开后进度丢失，或书籍一直加载打不开。

**可能原因**：epub.js ready 与位置恢复竞态；CFI 损坏。

**预防措施**：
- 每次翻页都保存进度（可加防抖）
- 加载时校验锚点，失败则回退到上次有效位置
- 章节超时 N 秒未加载则显示错误页
- 除 CFI 外，备用简单锚点（章节 + 偏移）

## 当前项目 Bug

### BUG-009：系统笔记本同步死循环（已修复 2026-07-23）

**现象**：导入学习数据 zip 一直「处理中」；底部 Tab 切换明显卡顿并露出过渡动画。

**原因**：`listNotebooks()` 每次全量同步词组/待补全词条并 `notifyNotebookDataChanged`；统计页监听该事件再调 `listNotebooks`，形成死循环。导入与切 Tab 均被拖死。旧备份格式本身无问题。

**修复**：列表仅确保系统笔记本存在；内容同步改为打开对应笔记本时懒执行；写盘默认静默不广播。

### BUG-008：章内阅读位置（暂缓）

**现象**：长章节需滚到底才能点 ⌂ 退出；章内滚动恢复不可靠。

**计划**：与 **视口分页** 一并解决（控制栏固定底部、按页记忆位置）。本轮不单独修补。
